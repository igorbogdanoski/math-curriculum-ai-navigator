### A2 — Батч генерирање на AI решенија

**Статус:** Готово за имплементација — **НЕ ПОЧНАТО** ⚠️  
**Файл:** `scripts/generate-ai-solutions.mjs`

**Пристап:**
1. Скриптата ги чита сите прашања од `matura_questions` каде `aiSolution = null`
2. За секое прашање повикува Gemini 2.5 Flash со структуриран prompt
3. Зачувува `aiSolution` назад во Firestore (merge)
4. Rate-limit aware: 5 барања/секунда, прескокнува веќе генерирани

**Формат на AI решение — MC прашања:**
```
**Постапка:**
Дадено: [услов на задачата]

Чекор 1: [прв чекор со математичка нотација]
Чекор 2: [втор чекор]
...

∴ **Одговор: А** — [кратко објаснување зошто другите избори се погрешни]
```

**Формат на AI решение — Отворени прашања (Part 2/3):**
```
**Стратегија:** [тип на задача — доказ на идентитет / пресметување / систем...]

Чекор 1: [постапка со LaTeX нотација]
Чекор 2: ...

**Проверка:** [валидација на решението]
∴ **Одговор:** [финален резултат]
```

**Очекувано траење:** ~10-15 мин за 622 прашања

## BLOC B — Студентски портал

### B1 — Student Login и профил

**Архитектура:** Без нов auth систем — го прошируваме постоечкиот Firebase Auth.

```
Firebase Auth (Google Sign-In)
        │
        ├── users/{uid}                    → TeachingProfile (наставници, role: teacher/admin)
        └── student_matura_profiles/{uid}  → StudentMaturaProfile (ученици, role: student)
```

**Нов тип `StudentMaturaProfile` во `types.ts`:**
```typescript
interface StudentMaturaProfile {
  uid: string;
  name: string;
  email?: string;
  photoURL?: string;
  track: 'gymnasium' | 'vocational4';
  examDate: string;          // '2026-06-06'
  createdAt: string;
  weakTopics: string[];
  practiceStats: {
    correct: number;
    total: number;
    byTopic: Record<string, { correct: number; total: number }>;
  };
  simulationCount: number;
  bestSimulationScore: number;
  streak: { count: number; lastDate: string };
}
```

**Auth flow:**
1. Корисникот отвора `/matura-portal`
2. Ако не е логиран: прикажи „Влези со Google" + „Продолжи без акаунт"
3. При Google Sign-In: провери дали постои `student_matura_profiles/{uid}` → ако не, создај нов профил (onboarding: **избери гимназиска/стручна патека**)
4. Прогресот се зачувува во Firestore (cross-device) ако е логиран, или `localStorage` ако не е

## Квалитетни стандарди за AI решенија

### Кога треба `aiSolution` (чекор по чекор)?
**СЕКОГАШ** — за сите 622 прашања. Видот на решението зависи од типот:

| Тип | Формат на решение |
|-----|-----------------|
| MC (Дел 1, 1 поен) | Кратка постапка + образложение зошто точниот одговор е точен + зошто грешните се погрешни |
| Open кратко (Дел 2, ≤4 поени) | Чекор-по-чекор со LaTeX, финален одговор |
| Open средно (Дел 2, 4-5 поени) | Целосна постапка, проверка, алтернативни методи |
| Open тешко (Дел 3, 5+ поени) | Целосна постапка + детален доказ + проверка + стратегиски коментар |

### Квалитетни критериуми
1. **Математичка точност** — проверени со CAS (WolframAlpha/Sage) за сложени задачи
2. **Педагошка јасност** — наменето за ученик 18-19 год., не за математичар
3. **LaTeX нотација** — сите математички изрази во правилна LaTeX нотација
4. **Детекција на заблуди** — за MC прашања, образложи зошто типичните грешки се грешки
5. **Јазик** — македонски, кириличен текст
