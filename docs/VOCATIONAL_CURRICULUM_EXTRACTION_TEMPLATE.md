# AI Studio — Шаблон за извлекување наставна програма (Средно стручно образование)

**Намена:** Го давате на AI Studio (или Gemini) заедно со PDF на официјалната МОН наставна програма.
Резултатот е JSON кој директно се вградува во `data/secondary/vocational4.ts` или `vocational3.ts`.

## ЗА 4-ГОДИШНО СТРУЧНО ОБРАЗОВАНИЕ

### PROMPT — Ставете го ова во AI Studio заедно со PDF-от

```

Твоја задача е да ја анализираш програмата и да вратиш структуриран JSON за секоја учебна година (X, XI, XII).

ВАЖНО — задолжително следи ги овие правила:

1. СТРУКТУРА — За секоја учебна година извлечи:
   - level: бројот на годината (X = 10, XI = 11, XII = 12)
   - Сите ТЕМИ со точниот назив од програмата
   - За секоја тема: бројот на часови, исходите на учење, содржините/поттемите
   - За секоја содржина: назив и краток опис (2-3 реченици)

2. ID КОНВЕНЦИЈА — задолжително следи ја:
   - Годишен ID: "voc4-grade-10", "voc4-grade-11", "voc4-grade-12"
   - Тема ID: "voc4-10-t1", "voc4-10-t2", ... (година-тема)
   - Концепт ID: "voc4-10-c1-1", "voc4-10-c1-2", ... (година-тема-концепт)

3. ЈАЗИК — сите текстови на МАКЕДОНСКИ (онака како стои во програмата)

4. ВРЕДНОСТИ — не измислувај; само она што пишува во документот

Врати го резултатот САМО како валиден JSON во следниот формат (без дополнителен текст, без markdown):
```

## ВЕРИФИКАЦИЈА — Дај го овој follow-up prompt

```
Провери го генерираниот JSON:

1. Дали се присутни СИТЕ теми од програмата (проверувај ги насловите)?
2. Дали збирот на часови по теми одговара на вкупните часови за таа година?
3. Дали секоја тема има барем 1 концепт?
4. Дали сите ID-а ги следат конвенциите (voc4-10-t1, voc4-10-c1-1, итн.)?
5. Дали "nationalStandardIds" е секогаш празна листа []?
6. Дали JSON-от е валиден (без trailing commas, без коментари)?

Ако постојат грешки, поправи ги и врати го целосниот коригиран JSON.
```

## ШТО ДА ПРАВИШ СО РЕЗУЛТАТОТ

### Чекор 1 — Зачувај го JSON-от
Во нов фајл (пр. `vocational4-raw.json`) во `data/matura/raw/` или директно.

### Чекор 2 — Конвертирај во TypeScript
Отвори `data/secondary/vocational4.ts` и замени ги stub-овите со реалните податоци.

Пример конверзија (JSON → TypeScript):
```typescript
// JSON:
// { "id": "voc4-grade-10", "level": 10, ... }

// TypeScript:
export const vocational4Grade10: Grade = {
  id: 'voc4-grade-10',
  level: 10,
  // ... (исто, само со TypeScript синтакса)
};
```

### Чекор 3 — Провери дали се компајлира
```bash
npx tsc --noEmit
```

### Чекор 4 — Commit
```bash
git add data/secondary/vocational4.ts
git commit -m feat-vocational4-real-curriculum-from-mon
```

## ЗАБЕЛЕШКИ

- **Ако програмата е иста за сите струки** (IT, економија, електро...): еден JSON e доволен, ID-ата се исти.
- **Ако струките имаат различни програми**: направи посебен JSON за секоја (и можеби посебен `.ts` фајл).
- **Ако некоја тема нема часови во документот**: ставете `"suggestedHours": 0` и забележете го тоа.
- **Ако исходите на учење ги нема во програмата**: напишете еден генеричен исход врз основа на содржините.
- **Математички поими** во описите не треба LaTeX — тоа е текстуален curriculum фајл, не испит.
