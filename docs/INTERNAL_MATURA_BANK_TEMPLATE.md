## Структура на базенот — препорачано за гимназиска матура

За 120 прашања, препорачана распределба:

| Тип | Број | Поени |
|-----|------|-------|
| MC (1 поен, DoK 1–2) | 80 | 80 |
| Отворени (4–5 поени, DoK 2–4) | 40 | 160–200 |

По тема:
- Алгебра и функции (X–XI): ~30 прашања
- Тригонометрија (XI–XII): ~15 прашања
- Exponенцијална/Логаритамска функција (XII): ~15 прашања
- Аналитичка геометрија (XII): ~10 прашања
- Низи, прогресии, граници (XIII): ~15 прашања
- Диференцијално сметање — извод (XIII): ~20 прашања
- Комбинаторика и веројатност (XII–XIII): ~15 прашања

## Workflow за внес од PDF

1. **Отвори PDF** и scaffold JSON паралелно во VS Code
2. За секое прашање, само пополни:

   | Поле | Опис | Задолжително |
   |------|------|:---:|
   | `questionText` | Текст + LaTeX `$...$` | ✅ |
   | `choices` | `{"А":...,"Б":...,"В":...,"Г":...}` или `null` | ✅ |
   | `correctAnswer` | `"А"` / `"Б"` / целото решение | ✅ |
   | `topic` | Слободен текст (`"Извод"`) | ✅ |
   | `topicArea` | Од табелата погоре | ✅ |
   | `conceptIds` | 1–2 ID-а од табелата | ✅ |
   | `points` | `1` за MC, `4` или `5` за отворени | ✅ |
   | `dokLevel` | 1–4 (ако не знаеш → `1`) | ⬜ |
   | `hints` | `[]` (се попол. подоцна) | ⬜ |
   | `aiSolution` | `null` (batch генерација подоцна) | ⬜ |

3. По секои 20–30 прашања → `git commit` (checkpoint)
4. По завршување: апликацијата го гледа автоматски — нема нов код
