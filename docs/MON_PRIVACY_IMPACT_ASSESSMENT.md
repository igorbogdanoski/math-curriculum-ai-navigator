## 1. Опис на обработката

### 1.1 Цел
Дигитална платформа за поддршка на наставата по математика за основно и средно образование во РСМ. Платформата овозможува:
- Креирање и споделување наставни материјали (планови, квизови, материјали)
- AI-помогнато генерирање содржина според националните стандарди (БРО)
- Проследување на напредокот на учениците (диагностика, IRT, misconception mining)
- Подготовка за државна матура

### 1.2 Категории на субјекти на податоци
| Категорија | Обем (estimate) | Тип на сметка |
|---|---|---|
| Наставници | ~200 (актуелно) → 5,000 (Y1) | Полна сметка |
| Ученици | 0 (актуелно) → 30,000 (Y1) | Поврзана со наставник |
| Директори / училишни администратори | ~20 (Y1) | Полна сметка |
| Родители (опционално) | ~1,000 (Y1) | Read-only портал |

### 1.3 Категории на лични податоци

**Идентификациски (наставник):**
- Е-пошта, име и презиме, профилна слика (опционално)
- Училиште / институција, предмет, одделенија

**Идентификациски (ученик):**
- Псевдо-ID (UID), име (опционално, само ако наставникот го внесе), одделение
- **НЕ собираме:** ЕМБГ, адреса, телефон, е-пошта (за малолетници под 13)

**Однесувачки / педагошки:**
- Резултати од квизови, време на одговор, прогрес
- IRT estimates (способност θ), DoK distribution
- Misconceptions detected by AI

**Технички:**
- IP адреса (само за rate-limiting, не се чува долгорочно)
- Web Vitals (LCP/CLS/INP/FCP/TTFB) — анонимизирани, агрегирани
- User-Agent (за device-type detection)

**НЕ собираме чувствителни категории** (член 9 GDPR): здравје, етничка припадност, политички ставови, верски убедувања.

## 6. Безбедносни мерки

### 6.1 Технички
- **Транспорт:** TLS 1.3 за сите конекции
- **Authentication:** Firebase Auth (bcrypt + scrypt) + опционен 2FA
- **Authorization:** Firestore Security Rules (role-based RBAC) + serverless function auth
- **Rate limiting:** UID 20/min + IP 100/min на AI endpoints
- **CSP:** Content Security Policy report-only во production
- **Шифрирање во мирување:** AES-256 на сите Google Cloud сервиси
- **Sentry release tagging:** За брза детекција на регресии

### 6.2 Организациски
- Пристап до production податоци: 1 admin (соло-developer)
- Логирање на admin акции
- Dependabot weekly + npm audit gate во CI
- Секундарни тестови: 1074 unit, ~20 E2E (Playwright)
- Pre-commit гејт: TypeScript strict, нула `as any`/`@ts-ignore`

### 6.3 Инцидент-менаџмент
- Sentry alerts → Slack/email
- Detection time target: < 5 мин
- User notification (breach): ≤ 72 часа согласно член 33 GDPR
- Контакт за инциденти: security@ai.mismath.net

## 7. Ризици и митигации

| Ризик | Веројатност | Удар | Митигација |
|---|---|---|---|
| Неавторизиран пристап до ученички податоци | Низок | Висок | RBAC + Firestore Rules + 2FA за наставници |

| Cross-site scripting (XSS) | Низок | Висок | DOMPurify + CSP + sanitizeHtml utility |
| Прекин на услуга | Низок | Среден | Vercel SLA 99.99% + status page |

## 8. DPIA заклучок

Согласно член 35 GDPR, оваа обработка вклучува систематско следење на однесувањето на децата (ученички напредок), што е означено како "висок ризик".

**Митигациите наведени во делови 5 и 6 се проценети како доволни** за намалување на резидуалниот ризик на прифатливо ниво.

**Овој PIA треба да се ревидира:**
- Годишно (следно: 25.04.2027)
- При секоја материјална промена на обработката
- При нов вид податок или нова трета страна
