## S113 — ЗАТВОРАЊЕ НА ЦИКЛУСОТ (Learning Loop)

> Цел: системот знае **кол ективно** колку одделението напредувало и го нуди тоа
> информативно ПРЕД наставникот да почне нов план — без да мора рачно да бара.

### S113-A — `hooks/useClassInsights.ts` (~120 LOC)

**Влез:** `teacherUid`, `grade`, `topicId` (опционално)
**Извор:** `quiz_results` колекција (постои, индексирана)

**Пресметува:**
- `avgScoreByTopic: Map<topicId, number>` — просечен % по тема, последни 30 дена
- `masteryRate: number` — % ученици со ≥85% на најмалку 1 прашање по темата
- `weakConcepts: string[]` — концепти под 60% (top 3)
- `recentTrend: 'up' | 'down' | 'flat'` — споредба со претходните 30 дена

**Query:** `quiz_results` where `teacherUid==` + `grade==` + `playedAt ≥ 30 дена`
→ limit(500) — постои composite index (`teacherUid ASC, playedAt DESC`)

```typescript
export function useClassInsights(
  teacherUid: string | null,
  grade: number | null,
  topicId?: string,
): { insights: ClassInsights | null; isLoading: boolean }
```

### S113-D — Scenario Bank квалитетен сигнал

**Проблем:** Рејтингот е субјективен. Нема сигнал кој сценарио **навистина работи**.

**Решение:** Корелација `lesson_executions.lessonPlanId` → `quiz_results.playedAt`

// services/firestoreService.scenarioBank.ts — нова функција
async function computeScenarioEffectiveness(scenarioId: string): Promise<{

наместо само ⭐⭐⭐⭐.

**Важно:** Само за сценарија со `sampleSize ≥ 10` — под тоа не се прикажува.

## S114 — ДЛАБОЧИНА НА AI ЛАБОРАТОРИИ

> Принцип: НЕ нови лаборатории. Постоечките добиваат **интерактивни задачи со
> следење и адаптивен feedback** — трансформирајќи ги од визуелизации во
> вежбалишта.

### Зошто длабочина, не ширина

Моментално лаборатриите се **пасивни**: наставникот ги користи за демонстрација,
но ученикот не интерагира со нив структурирано. Нема:
- Чекор-по-чекор решавање со hint систем
- Следење на напредокот во лабораторијата
- Врска со `quiz_results` / `concept_mastery`

Уште 3 нови лаборатории без ова = уште 3 демо алатки. Со ова = учење.

### S114-A — `hooks/useLabSession.ts` (~90 LOC) — универзален hook

```typescript
interface LabSession {
  labId: string;         // 'trig' | 'numtheory' | 'placevalue' | ...
  studentName: string;
  exercisesCompleted: number;
  correctAnswers: number;
  hintsUsed: number;
  durationSeconds: number;
}
// Зачувува во quiz_results со quizType: 'lab_session'
// Ова ги поврзува лаб активностите со постоечкиот прогрес систем
```

### S114-B — Продлабочување по лаборатории

**TrigonometryLab** (тековно: unit circle + wave + identities)
- Додај: **Чекор-по-чекор решавач** — наставникот внесува равенка (sin²x + cos²x = ?),
  системот го води со hints
- Додај: **Exam mode** — 5 прашања генерирани по тип (идентитети / вредности / равенки),
  резултатот оди во `useLabSession`
- Длабочина: Proof Explorer — покажи ги изводите на идентитетите интерактивно

**NumberTheoryLab** (тековно: primes/sieve, GCD/LCM, modular, sequences)
- Додај: **Интерактивно Евклидово алгоритмо** — чекор-по-чекор со визуелизација
- Додај: **Prime factorization tree** — кликабилно дрво за разложување
- Додај: **Криптографија** — едноставен Caesar cipher → RSA основи за постари одд.

**PlaceValueLab** (тековно: Прикажи/Вежбај/Состави)
- Додај: **Собирање со пренос** — анимирано пренесување меѓу колони (Dienes блокови)
- Додај: **Споредување броеви** — "<" ">" "=" со блок визуелизација
- Додај: **Различни бројни бази** (бинарна, осмична) за постари одделенија

**ProbabilityLab** (тековно: дрво на веројатности)
- Додај: **Симулатор** — фрли монета N пати, прикажи конвергенција кон теоретска
- Додај: **Условна веројатност** — Venn дијаграм интерактивен
- Додај: **Байес визуелизација** — за гимназиски ниво

**Geometry3DLab** (тековно: калкулатор за тела)
- Додај: **Интерактивна 3D ротација** (Three.js или WebGL — евалуирај bundle size)
- Алтернатива без 3D lib: **анимирани SVG проекции** (полесно, нема bundle impact)
- Додај: **Чекор-по-чекор волумен** — не само резултатот, туку формулата распишана

### S114-C — Lab-to-Quiz bridge

Кога ученикот завршува лаб сесија (преку `useLabSession`), системот нуди:
"Провери го своето знаење → [Стартувај квиз за оваа тема]"
→ постоечкиот quiz flow со `prefillConceptId` параметар.

Ова го затвора: Лаб (разбирање) → Квиз (проверка) → Прогрес (следење).

## S115 — РОДИТЕЛСКИ/УЧЕНИЧКИ АКАУНТИ

### Реална проценка на трошокот

Ова НЕ е мала фича. Честен преглед:

#### Техничка сложеност

| Компонента | Тековно | Потребно | Estimate |
|------------|---------|----------|----------|
| Auth | Firebase email/password (наставници) | + ученик auth + родител auth | +2 недели |
| Безбедносен модел | teacherUid-based | 4 роли × N правила | +1 недела |
| Firestore Rules | ~80 правила | ~150+ правила | +1 недела |
| Ученички UI | `/#/portfolio?name=` (fragile) | Proper authenticated view | +2 недели |
| Родителски UI | Нема | Dashboard: дете → резултати → портфолио | +2 недели |
| Class roster | Нема (name-based) | Teacher manages students list | +1 недела |
| GDPR за малолетници | Не постои | Consent flow, data deletion, parental consent | +1 недела |
| **Вкупно** | | | **~10 недели** |

#### Firebase трошок (реален)

```
Тековно (450 наставници):
  Firestore reads: ~50,000/ден → $0.03/ден → ~$1/месец

Со ученички акаунти (450 наставници × 25 ученици = 11,250 ученици):
  Ако секој ученик прави 20 reads/ден: 225,000 reads/ден → ~$0.14/ден → ~$4/месец
  Со portfolio генерација (AI): +Gemini API повици × 11,250 → значително повеќе
  Realtime listeners за сите ученици: потенцијален spike
  
Worst case со активни ученици: $50-150/месец (наспроти $1-5 денес)
```

#### GDPR специфики за Македонија

- Закон за заштита на лични податоци (ЗЗЛП) — 2020, базиран на GDPR
- Малолетници под 14 години: **задолжителна родителска согласност** за секое собирање на податоци
- Задолжително: право на бришење, преносливост, ограничување на обработка
- Потребен: Офицер за заштита на лични податоци (ДПО) или изјава дека не е потребен
- **Ризик:** Школото/наставникот стануваат Data Controller — правна одговорност

#### Препорака: Фазиран пристап наместо big bang

**Фаза 0 (S115-A) — Веднаш, без нов auth:**
Подобри ја тековната `?name=` патека:
- `localStorage` → `sessionStorage` per-tab (помала прецизност на делење)
- QR код за ученикот кој директно го отвора неговото портфолио (generate во ClassroomView)
- PIN-based пристап (4 цифри + ime) наместо само ime

**Фаза 1 (S115-B) — Opt-in ученички email:**
- Наставникот внесува e-mail листа за класата
- Firebase `createUserWithEmailAndPassword` за секој ученик
- Само `student` роља — read-only на сопствени `quiz_results`
- Без родителски пристап уште

**Фаза 2 (S115-C) — Родителски пристап:**
- Parent invite link (токен со TTL)
- Само read: портфолио + резултати на детето
- Согласност форма (GDPR)
- **Дискутирај со правник пред имплементација**

**Кога да започне Фаза 1:** Кога ќе имаш ≥2,000 активни наставници.
Под тоа прагот, трошокот/сложеноста не се оправдани.
